## What to backup?

If you are a PRO user, ensure you have a backup of your subscription key.

There are also several elements you need to backup:

:::warning[Please Note]
Many docker users use __-ti__, __-it__ or __-t -i__ arguments to execute commands on docker containers. To get reliable backups on docker, please use only __-i__, as __-t__ will create a pseudo-tty and make your backup files unusuable.
:::

### 1. The database

This can be easily scripted using mariadb-dump. Use docker compose to connect to the database container and write mariadb-dump output to a local file.

Be sure to use simple-quotes for the bash -c argument to be able to use __MYSQL_USER__, __MYSQL_PASSWORD__ and __MYSQL_DATABASE__ environment variables.

<Tabs groupId="edition">
  <TabItem value="ce" label="CE" default>
    <CodeBlock>
      {`docker compose -f docker-compose-ce.yaml exec db bash -c \\
'mariadb-dump -u\${MYSQL_USER} -p\${MYSQL_PASSWORD} \${MYSQL_DATABASE}' \\
> /path/to/backup.sql`}
    </CodeBlock>
  </TabItem>
  <TabItem value="pro" label="PRO">
    <CodeBlock>
      {`docker compose -f docker-compose-pro.yaml exec db bash -c \\
'mariadb-dump -u\${MYSQL_USER} -p\${MYSQL_PASSWORD} \${MYSQL_DATABASE}' \\
> /path/to/backup.sql`}
    </CodeBlock>
  </TabItem>
</Tabs>

### 2. The server public and private keys

You can use docker compose cp to backup the Passbolt GPG keys:

<Tabs groupId="edition">
  <TabItem value="ce" label="CE" default>
    <CodeBlock>
      {`docker compose -f docker-compose-ce.yaml cp passbolt:/etc/passbolt/gpg/serverkey_private.asc \\
    /path/to/backup/serverkey_private.asc`}
    </CodeBlock>
    <CodeBlock>
      {`docker compose -f docker-compose-ce.yaml cp passbolt:/etc/passbolt/gpg/serverkey.asc \\
    /path/to/backup/serverkey.asc`}
    </CodeBlock>
  </TabItem>
  <TabItem value="pro" label="PRO">
    <CodeBlock>
      {`docker compose -f docker-compose-pro.yaml cp passbolt:/etc/passbolt/gpg/serverkey_private.asc \\
    /path/to/backup/serverkey_private.asc`}
    </CodeBlock>
    <CodeBlock>
      {`docker compose -f docker-compose-pro.yaml cp passbolt:/etc/passbolt/gpg/serverkey.asc \\
    /path/to/backup/serverkey.asc`}
    </CodeBlock>
  </TabItem>
</Tabs>

### 3. The application configuration
Passbolt docker stores its configuration as environment variables.

If you are using docker compose, environment variables are on the env folder:

* env/passbolt.env
* env/mysql.env

If you are running docker container, you should have passed these variables through the command line.
Please check the [passbolt env variable reference](/hosting/configure/environment-reference/)
